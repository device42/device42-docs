Device Ignore Rules let you exclude specific devices from discovery jobs. This is useful for devices you don't need in the database, such as network printers, fax machines, or IP phone systems.

Ignore rules do not affect data that has already been discovered. They only apply to discovery jobs you run after the rules are created, and they work for all discovery job types, including cloud discovery.

You can view ignore rule results on the [Discovery Scores](/reports/reports/discovery-quality-scores.mdx) page (**Reports > Discovery Scores**).

:::info
Ignore rules **skip** device discovery completely. If you want to discover devices but exclude specific IPs, MACs, or network interfaces from the dataset, see [Windows and Hyper-V Autodiscovery](/auto-discovery/windows-and-hyper-v-auto-discovery/#option-to-ignore-ipsmac-addresses).
:::

## Add a Device Ignore Rule

Select **Tools > Settings > Device Ignore Rules** and then click **Create** to apply a Device Ignore Rule to a device.

<ThemedImage
  alt="Device Ignore Rule options"
  sources={{
    light: useBaseUrl('/assets/images/device-ignore-rules/device-ignore-dropdown-light.png'),
    dark: useBaseUrl('/assets/images/device-ignore-rules/device-ignore-dropdown-dark.png'),
  }}
/>

Select an **Ignore device based on** option:

- **Device name**
- **Device type**
- **OS name**
- **Mac address prefix**
- **Hardware model**
- **Tag** (device tag)

Enter a string in the **Ignored text** field. If the string matches any portion of the corresponding attribute, that device is ignored. All devices with matching attributes are ignored.

You can refine the rule with **Text length match type** (**Partial match** or **Exact match**) and **Text case match type** (**Case insensitive** or **Case sensitive**).

The page also displays a **Last time** timestamp if the rule has already been triggered during a discovery job.

If you select **Device type** for the **Ignore device based on:** option, Device42 displays additional fields you use to select the **Device Type** and **Subtype**.

## Add Device Ignore Rules from a Device List

You can also add ignore rules directly from a device list page. Navigate to any device list (for example, **Resources > Compute > All Devices**), select the devices you want to ignore, and choose **Ignore devices from autodiscovery** from the **Actions** menu.

<ThemedImage
  alt="Ignore devices action"
  sources={{
    light: useBaseUrl('/assets/images/device-ignore-rules/ignore-devices-light.png'),
    dark: useBaseUrl('/assets/images/device-ignore-rules/ignore-devices-dark.png'),
  }}
/>

The **Ignore devices from autodiscovery** page lists the devices you selected. Use the **Choose device attribute** dropdown to select the attribute to base the rules on.

<ThemedImage
  alt="Configure rule"
  sources={{
    light: useBaseUrl('/assets/images/device-ignore-rules/ignore-devices-options-light.png'),
    dark: useBaseUrl('/assets/images/device-ignore-rules/ignore-devices-options-dark.png'),
  }}
/>

Select the **Choose case match type attribute** (**Case insensitive** or **Case sensitive**) and **Choose length match type attribute** (**Partial match** or **Exact match**) to refine the rule.

Click **Create new device ignore rules** to add the rules.
