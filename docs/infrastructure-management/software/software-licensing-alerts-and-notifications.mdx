Software licensing alerts notify you by email or Device42 notification — or send data to a third-party application — when a software condition you define is met. Triggered alerts are called notifications, and you can manage them to communicate resolution status to your team.

## Create Alerts

Set up a software alert by navigating to **Analytics > Setup Alerts** and selecting **Software** as the **Rule Type**.

Three **Trigger** types are available for software licensing. You can create a rule based on:

- **Count**: The absolute number of licenses.
- **License Percent Used**: The percentage of available licenses used.
- **Prohibited Software Installed**: Whether prohibited software has been installed on a machine.

When a rule is triggered, emails are sent to the address(es) specified under the **Escalation Profile** option. In this example, when more than 85% of the licenses for a software component have been used, the members of **Alert Group 1** are notified by email. If a profile is already configured, you can select it from the prepopulated list. If not, click the **plus icon** next to the **Escalation Profile** field to open a form to configure one.

Select the **Apply To: All** checkbox to apply the alert to all software components, or search for specific software under **Available objects** and move them to **Chosen objects** using the **right arrow** button.

## Manage Notifications

Each triggered alert rule creates a notification. Notifications for all rule types are listed under **Analytics > Notifications**.

Only one email and one notification will be sent per software component per trigger.

### Bulk Actions

To perform a bulk action, select notifications from the table, choose an action from the **Actions** dropdown, and confirm the dialog:

- **Delete**: Choose whether to delete the selected notifications immediately or only after confirmation. If the condition recurs, a new alert and notification will be created.
- **Export Selected Columns as CSV**: Create a CSV file with the selected notifications.
- **Acknowledge Notification**: Use this bulk action and the corresponding **Acknowledged** column when multiple system administrators receive the alert. One system administrator can then acknowledge the alert to indicate that they are working on the issue.
- **Clear Conditions**: Indicates to Device42 that a condition has been fixed. A new alert and notification will be created if the condition recurs. Each cleared action under **Cleared** will have a green checkmark. Only one notification will be sent until the condition is marked as cleared.
