This page explains how to migrate existing power discovery jobs to the new Power RC SNMP discovery using the imports/exports tool.

## Prerequisite

Set up a Remote Collector before proceeding. See [Remote Collector - Installation](/getstarted/deploy-device42/remote-collector-rc-installation.mdx) for instructions.

## Migrate Existing Power Appliance Jobs to Power RC

1. Navigate to **Tools > Import/Export (XLS)**.

    ![Export current power discovery jobs](/assets/images/import-export-xls.png)

2. Export the current data for **Add Power Autodiscovery Jobs**.

    ![Download Current Data](/assets/images/download-current-data.png)

3. In the exported file, update cell A1 to contain `/api/1.0/auto_discovery/networks`.

    ![Modify data in cell A1](/assets/images/power-xls-config.png)

4. Rename the header in column V from `appliance` to `remote_collector_id`.

    ![Rename header in column V](/assets/images/power-xls-column-rename.png)

5. Delete all data under the column V header and replace it with Remote Collector IDs. To find the IDs, navigate to **Tools > Autodiscovery > Remote Collectors**.

    ![Select remote collector to view](/assets/images/select-rc-view.png)

6. Import the file from the same **Tools > Import/Export (XLS)** page.

    ![Re-import the modified XLS on the import/export page](/assets/images/import-modified-xls.png)

7. After a successful import, the jobs will no longer appear under **Tools > Autodiscovery > Power (SNMP)**. Navigate to **Tools > Autodiscovery > SNMP** and edit the jobs to enable monitoring.

    ![Jobs moved after successful import](/assets/images/enable-monitoring-discovered-pdu.png)
