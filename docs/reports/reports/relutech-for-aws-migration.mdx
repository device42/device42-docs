Relutech and Device42 have teamed up to ease migrations to AWS. Device42's deep discovery provides Relutech with the information required to price your on-premises physical assets for purchase and leaseback, as well as third-party maintenance. Relutech then purchases those assets and provides them back to the customer on a leasing schedule. As you migrate those workloads to AWS, the workload is then rolled off of your lease, thereby reducing your on-premises costs as you increase consumption in AWS.

## Set Up Discovery Jobs

The recommended approach to capturing your physical (bare metal) server inventory involves three types of discovery jobs, run in the following order: OS-level scans first, then SNMP scans against the BMC, then the Warranty sync. Each job type builds on the data collected by the previous one.

## Hypervisor/*nix/Windows Job Creation

Set up [Hypervisor / *nix / Windows](/auto-discovery/linux-unix-server-auto-discovery) discovery jobs for each type of physical device. These OS-level scans create the device record and capture parts including CPU and HBA card information.

- For the Windows and *nix jobs, ensure that **Discover Parts** is checked in the **Host Discovery** section.
- For Hypervisor jobs, **Discover Parts** is enabled by default and cannot be changed.

## SNMP Job Creation

Create [SNMP](/auto-discovery/storage-arrays-autodiscovery/snmp-san-server-auto-discovery) discovery jobs to scan the management IPs of your targeted servers. These jobs augment the OS-level data by adding management MACs, IPs, and additional parts such as RAM, disks, and PSUs.

- Enter the IP or FQDN of all the devices that the credentials and access apply to under **Server(s)**.

## Warranty Job Creation

Depending on the vendor, warranty jobs retrieve service contracts associated with each server. If the vendor system is unavailable, warranty data can be added manually via the UI or spreadsheet imports.

See the [Warranty Sync](/auto-discovery/warranty-autodiscovery) documentation for setup instructions.

## Recommended Automated Scheduling

Set up automated schedules to keep your inventory data current for the Relutech report.

| Job type | Recommended schedule |
|----------|---------------------|
| Hypervisor/\*nix/Windows | **Daily** |
| SNMP | **Weekly** |
| Warranty Sync | **Weekly** |

## Run the Relutech Report

After the discovery jobs have been completed, navigate to **Insights+** to extract the data for Relutech. The Relutech report is located under **Advanced Reporting** > **Integrations** > **IT Asset Disposition** > **Relutech**.
